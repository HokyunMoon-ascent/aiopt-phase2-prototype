**EntityLoop**

# **목차**

# **1. 배경 & 문제 정의**

검색의 소비 지점이 ‘10개의 파란 링크’에서 AI 답변(ChatGPT·Perplexity·Google AI Overviews·Gemini)으로 이동하고 있다. AI는 페이지를 키워드가 아니라 개체(entity)와 그 관계·구조로 이해하고, 소비자의 질문을 잘게 분해해 가장 잘 커버·구조화된 출처를 인용한다.

그 결과 브랜드는 ‘검색 결과 안’이 아니라 ‘AI의 답변 안’에서 경쟁한다. 광고와 콘텐츠를 집행해도, 소비자가 상황을 설명하며 던진 질문의 AI 추천 후보군에 들지 못하면 브랜드는 소비자에게 닿지 못한다. 핵심 문제는 두 가지다:

- ‘사람들이 무엇을 어떤 의도로 검색하는가(수요)’와 ‘AI가 내 브랜드를 어떻게 읽는가(인식)’가 분리되어, 진짜 격차를 자동으로 잡지 못한다.

- 브랜드가 ‘의도한 좌표’(우리가 콘텐츠에 적어둔 정체성)와 AI가 ‘실제로 읽은 좌표’(시장 정보 흔적에서 합성한 인식)가 어긋난다 — 이것이 엔티티 갭이다.

**근거 자산:** 이 도구는 Ascent의 리스닝마인드(검색 수요, 결정론적 실데이터)와 Ontologizer(개체·구조화) 로직, 그리고 도서 _《AI 브랜드 마케팅》_ 4·5장의 엔티티 갭 운영 프레임을 결합한다.

# **2. 목표 & 비목표**

### **목표(Goals)**

- 키워드/브랜드/CEP 입력만으로 ‘AI가 내 브랜드를 어떻게 인식하는가’를 실제 AI 호출로 측정한다.

- 엔티티 갭을 5유형(카테고리·속성·관계·CEP·신뢰)으로 진단하고, 갭별 보강·담당 조직·재측정 신호까지 실행안을 산출한다.

- 모든 권고는 추측이 아니라 두 소스(리스닝마인드·실제 AI 응답)의 근거를 가진다.

- 로컬에서도, 외부 웹서버에서도 바로 설치해 쓸 수 있도록 패키지로 배포한다.

### **비목표(Non-Goals)**

- 콘텐츠 자동 발행, 백링크·도메인 권위 풀스택 SEO 감사는 범위 밖.

- 리스닝마인드 알고리즘 자체의 재구현은 범위 밖(API 소비).

# **3. 핵심 개념**

### **3.1 CEP (Category Entry Point, 카테고리 진입점)**

소비자가 특정 카테고리로 들어오는 상황·계기·목적·불편·감정·제약의 묶음. 측정 단위는 브랜드 전체가 아니라 CEP다. 예: ‘면역력 챙기려고 매일 먹기 좋은 멀티비타민을 찾는 상황’.

### **3.2 엔티티 갭 (Entity Gap)**

브랜드가 의도한 인식(온드 미디어에 정리한 좌표)과 AI가 실제 답변에서 합성한 인식(시장 정보 흔적에서 읽은 좌표) 사이의 차이. 핵심은 이 ‘AI 인식’을 실제로 AI에게 물어 측정한다는 점이다.

| 갭 유형     | 무엇이 어긋났나                               | 보강 방향                            |
| :---------- | :-------------------------------------------- | :----------------------------------- |
| 카테고리 갭 | AI가 잘못된/너무 넓은 범주로 읽음             | 브랜드 정의·카테고리 언어 정비       |
| 속성 갭     | 핵심 구매요인(KBF)이 답변에 안 나옴           | 상세/FAQ/리뷰에 ‘조건·근거 문장’으로 |
| 관계 갭     | 경쟁·비교군 내 위치·비교축이 어긋남           | 비교 기준·선택 이유 구조화           |
| CEP 갭      | 특정 상황 질문에서 호출 안 됨                 | 그 상황 직접 다루는 콘텐츠·리뷰      |
| 신뢰 갭     | 온드 주장 vs 외부 신뢰 신호 간극(최신성 포함) | 미디어·전문가·커뮤니티·리뷰 강화     |

### **3.3 4대 KPI (AI 응답 분석)**

- 브랜드 언급(Brand Mention) — 넌브랜디드 프롬프트에서 등장 여부·순위

- 콘텐츠 인용(Contents Citation) — AI가 근거로 삼은 출처(자사 vs 외부)

- 브랜드 긍부정(Brand Sentiment) — 답변 안에서의 긍/중/부 맥락

- AI 유입 트래픽(AI Traffic) — AI 답변을 통한 실제 유입(후행 지표)

# **5. 시스템 아키텍처**

**Claude를 엔티티 엔진으로:** 의미 추론(엔티티 추출·정의·타입·관계·CEP 도출·응답 분석)은 Claude가, 데이터 수집·KG 연결·점수·렌더·저장은 Python이 담당한다. 엔진은 로컬 claude CLI 또는 ANTHROPIC_API_KEY(REST)로 동작하므로, CLI가 없는 외부 서버에서도 키만 있으면 작동한다.

| 모듈                            | 역할                                                                        |
| :------------------------------ | :-------------------------------------------------------------------------- |
| lmpipe_client / gather          | 리스닝마인드 /v1/analyze 호출 → 수요 번들(클러스터·콘텐츠갭·SERP·신뢰 가드) |
| daas                            | ListeningMind DaaS REST — cluster/path/intent/keyword 파인더                |
| ontologizer / kg                | 개체 추출·salience + Wikidata/Wikipedia 식별자(sameAs) 연결                 |
| engine / engines                | Claude(CLI·API) 엔진 + OpenAI/Gemini/Perplexity 어댑터                      |
| geo_audit                       | CEP→프롬프트 포트폴리오→실제 AI 호출→4 KPI·5유형 갭 진단·보강 액션          |
| blueprint / features / matching | 엔티티 온톨로지·F1 갭·F2 AEO 점수·JSON-LD                                   |
| store                           | 세션·결과 영속(저장/이력/트렌드/다운로드 원본)                              |
| web / cli                       | 웹앱(2탭·비동기 잡·히스토리·다운로드) · CLI(서브커맨드)                     |

# **7. 외부에서 사용하는 방법 (배포)**

EntityLoop은 표준 라이브러리만으로 동작(zero-dependency)하며, 외부 서버엔 claude CLI가 없으므로 ANTHROPIC_API_KEY로 엔진을 돌린다.

| 방식           | 방법                                                                  | 외부 접근    |
| :------------- | :-------------------------------------------------------------------- | :----------- |
| 로컬           | python3 serve.py → localhost:8765 (claude CLI 로그인 사용)            | 내 PC 전용   |
| 빠른 외부 공유 | 로컬 실행 + ngrok/Cloudflare Tunnel로 임시 공개 URL 발급              | 임시 URL     |
| VPS/서버       | pip install (휠) 후 entityloop-web --host 0.0.0.0 --port 8765         | 서버 IP:포트 |
| 도커           | cp .env.example .env → docker compose up -d (영속 볼륨·헬스체크 포함) | 서버 IP:8765 |
| 클라우드 PaaS  | Render/Fly/Railway에 도커 배포 — PORT 자동 인식                       | https URL    |

### **공개 노출 시 권장 설정**

- **ENTITYLOOP_TOKEN** 설정 → 접속에 ?token=… 필요(간단 인증). 페이지가 토큰을 모든 요청·다운로드에 자동 전파.

- **ANTHROPIC_API_KEY** (엔진) · **LM_API_KEY** (DaaS) · **MAISTRO_API_KEY** 를 .env 또는 환경변수로.

- **GET /healthz** 로 엔진·DaaS 활성 여부 확인. 세션은 **ENTITYLOOP_DATA** 경로(영속 볼륨)에 저장.

**한 줄 배포:** cp .env.example .env (키 채우기) → docker compose up -d → http://서버:8765

# **8. 산출물 & 검증**

- 패키지: pip 설치 가능(콘솔 명령 entityloop · entityloop-web), 휠/sdist 빌드·설치 검증 완료(v0.3.2).

- DaaS·lmpipe·CEP 발굴·GEO Audit·히스토리·다운로드 모두 라이브 엔드투엔드 검증.

**대표 사례:** ‘아임비타 · 멀티비타민 · 면역 CEP’ 실측에서 넌브랜디드 프롬프트 4개 전체 브랜드 언급률 0%, 그리고 AI가 모기업을 ‘동아제약’으로 오인식(실제 일동제약)하는 신뢰·관계 갭을 포착. 이는 페이지 본문 분석으로는 원리적으로 발견 불가하며, ‘AI에게 실제로 물어’ 측정하는 본 도구의 차별점이다.
