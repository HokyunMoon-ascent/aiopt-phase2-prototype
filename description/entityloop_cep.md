# 카테고리에서 CEP 자동 발굴은 어떻게 실행되나

## 한 문장 요약

**"카테고리 이름 하나만 넣으면 → 실제 검색 수요 데이터를 근거로 → AI가 '소비자가 이 카테고리에 발을 들이는 상황(CEP)' 후보 8개를 뽑아주는" 기능입니다.**

여기서 CEP(Category Entry Point)란 검색어 목록이 아니라 **소비자가 처한 구체적인 장면**입니다.

예를 들어 "영양제" 카테고리라면 —

- ❌ "비타민D", "유산균" (그냥 검색어)
- ✅ "환절기마다 감기를 달고 사는 아이 때문에 뭘 먹여야 할지 고민되는 엄마" (상황·불편·감정이 묶인 장면)

---

## 실제로 일어나는 일 (4단계)

### 1단계 — 시장의 진짜 수요를 긁어옵니다

카테고리 이름을 넣으면 리스닝마인드(검색 데이터 분석 도구)에 물어봐서, 사람들이 실제로 뭘 검색하는지 · 어떤 고민 덩어리(클러스터)가 있는지 · 아직 아무도 콘텐츠로 안 채운 빈틈은 뭔지를 모아옵니다.

→ **상상이 아니라 실제 데이터가 출발점**입니다.

### 2단계 — AI에게 줄 근거 자료를 추립니다

1단계에서 모은 방대한 데이터 중 가장 중요한 것들(수요 상위 40개, 고민 덩어리 12개 등)만 간추립니다. 여기에 더 깊은 소비자 인식 데이터(DaaS)가 연결돼 있으면 근거를 한 겹 더 보탭니다. (없어도 그냥 진행)

### 3단계 — AI가 '상황'으로 번역합니다

간추린 데이터를 AI(Claude)에게 주면서 이렇게 지시합니다:

> "이건 검색어 나열이야. 이걸 근거로, 소비자가 이 카테고리에 진입하는 **구체적 장면** 8개를 뽑아줘. 특히 **아직 어떤 브랜드도 제대로 해결 못 해서 경쟁이 덜한 '숨은 불편'**을 우선으로."

### 4단계 — 결과물이 나옵니다

각 CEP마다 이런 항목이 채워져 나옵니다:

- **이름** — 짧은 라벨
- **상황** — 한 문장
- **관심도** — 높음 / 중간 / 낮음
- **불편 유형** — 명시적 불편인지 / **숨은 불편**인지
- **관련 검색어** — 몇 개
- **질문 예시** — 소비자가 AI에게 실제로 물어볼 법한 질문

---

## 흐름 도식

```
카테고리 입력 ("영양제")
   │
① 시장 수요 수집        ← 리스닝마인드에서 실제 검색 데이터
   │   (검색어 · 고민 덩어리 · 콘텐츠 빈틈)
   │
② 근거 자료 추리기      ← 핵심만 간추림 (+ DaaS 있으면 보강)
   │
③ AI가 '상황'으로 번역  ← 검색어 → 소비자 장면(CEP)
   │   ("숨은 불편" 우선)
   │
④ CEP 후보 8개 출력     ← 이름 · 상황 · 관심도 · 검색어 · 질문 예시
```

---

## 왜 이렇게 만들었나 (핵심 가치)

1. **데이터 기반** — 담당자의 감이 아니라 실제 검색 수요에서 출발합니다.
2. **"숨은 불편" 우선** — 남들 다 아는 경쟁 치열한 상황 말고, **아직 빈 시장**을 먼저 찾아주는 게 이 기능의 진짜 차별점입니다.
3. **다음 작업의 재료** — 여기서 나온 CEP는 그다음 단계("우리 브랜드가 이 상황에서 AI 답변에 잘 등장하는가?"를 진단하는)로 바로 넘어가는 입력이 됩니다.
